- **Task 1** — `sim_core` 작업 사이클(픽업→운반→배치) + `tick()`/`plan_robot`/`safe_plan_robot`/`safe_tick`에 `conveyor_running` 전파 + `work_points`(순찰 지점 재사용 대신 그리드 전체 해시 분산) + `TriggerArmAction` 악용 경로 차단(task를 입력으로 안 읽고 파생만 시킴, 뮤테이션 테스트로 실증). 코드 품질 리뷰에서 카운트다운 타이밍이 정확히 검증되는지(1틱 일찍 끝나는 회귀를 잡는지) 지적받아 테스트를 `PICK_TICKS-1`/`PLACE_TICKS-1` 체크포인트까지 강화.
- **Task 2** — 생산량 집계를 "컨베이어 켜짐+로봇 존재"라는 전제에서 "실제 배치 완료"(`detect_completed_placements`)로 교체. 코드 품질 리뷰에서 REST 통합테스트의 `> 0.0` 단언이 과다 집계 회귀를 못 잡는다는 게 발견돼(뮤테이션 테스트로 3 vs 150 확인), 그리드/틱 상수 기반 상한 단언을 추가해 실제로 막도록 강화.
- **Task 3** — `RobotView.carrying` 와이어 필드(Task 2에서 컴파일을 위해 선반영됨) + WS 통합테스트로 실제 전이 검증.
- **Task 4** — 클라이언트 화물 아이콘 렌더링. 코드 품질 리뷰에서 새 E2E 테스트가 "결국 나타남"만 확인하고 "처음엔 없어야 함"은 확인 안 한다는 게 발견돼(뮤테이션 테스트로 확인), 픽업 완료 전 부재를 먼저 단언하도록 강화 — 그 과정에서 캔버스 rAF 그리기 루프와 DOM 갱신 사이의 실제 타이밍 경쟁도 발견해 고쳤다.
- **Task 5** — 전체 검증 + 문서 갱신. `cargo test --manifest-path server/Cargo.toml` 151/151 통과(59 lib + 68 main + 3 pathfinding + 8 REST 통합 + 2 정적 서빙 통합 + 5 tick 속성 + 6 WS 통합), `cargo clippy --manifest-path server/Cargo.toml --all-targets`는 이 플랜과 무관한 기존 `sim.rs:363`의 unknown-lint 경고 1건 제외 0개, `npm run typecheck` 에러 없음, `npm test`(vitest 단위 64/64, 10개 파일 — 이 플랜은 새 유닛테스트 파일을 추가하지 않음), `npm run test:integration`(3/3), `npm run build` 성공, `npx playwright test`(3/3 — Task 4가 추가한 화물 아이콘 테스트로 2개에서 3개로 증가). Playwright 전체 스위트를 첫 실행에서 화물 아이콘 테스트 1건이 실패했다가(픽업 완료 판정 직후 rAF 그리기와 DOM 상태 사이의 타이밍 경쟁으로 추정) 곧바로 이어진 격리 재실행 4회 + 전체 스위트 재실행 3회 전부 통과(7/7)로 재현되지 않는 플레이키니스임을 확인 — 이 플랜이 만든 새 회귀가 아니라 Task 4에서 이미 발견/완화됐던 것과 같은 종류의 타이밍 경쟁이 완전히 제거되지는 않은 것으로 보이며, 후속 관찰이 필요하면 Backlog에 남긴다.

**목적있는 이동(작업 사이클) 전체 완료.** 라이브 데모에서 로봇이 순찰만 돌고 실제 "일"을 하지 않는다는 피드백을, 컨베이어 on/off에 연동된 픽업→운반→배치 작업 사이클로 해소했다. 생산량 통계가 처음으로 실제 배치 완료 이벤트에 연결됐고, 오퍼레이터의 수동 팔 조작이 자동 사이클 로봇에게는 안전하게 무력화되도록 설계 단계에서부터 악용 경로를 차단했다.
